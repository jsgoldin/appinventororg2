Build the NoteTaker App


Note-Taker is a simple app for storing notes-- workout records, to-do lists, anything you might want to jot down. It is a good model for building apps with user-generated data using list variables and the TinyDB database.

Build It
Follow the video steps below to build the app yourself.

Rather use the book chapter text? Download the PDF 



step 1: design the input form  Create an input form with label, button, and a hint (instructions).

Looks best in full screen.

step 2: add entered items into a list

Define a list variable with no items to start. When the user clicks submit button, add the entered note to the list, then show the list in the NotesListLabel and empty out the text box.

Looks best in full screen.

step 3: display the list items on separate lines

As each item is entered, add a new line character and the new item to the label displaying the list.

Looks best in full screen.

step 4: add a timestamp and display with last note on top

Add a Clock component, then use the Clock.Now to get a timestamp. Re-order the way you modify the label displaying the list.

Looks best in full screen.

step 5: store the data persistently and retrieve on startup

Add a TinyDB component, then store the list in the database each time a new item is entered. When the app opens (Screen.Initialize) retrieve the data, making sure to check that there is indeed a list in the database (not first time the app has been run).

Looks best in full screen.

step 6: refactor the blocks by introducing a DisplayList procedure

Add a procedure and move the foreach display code from Screen.Initialize into it. In SubmitButton.Click, call this procedure instead of the different display code.

Looks best in full screen.

step 7: provide a delete item button

Add a ListPicker component that allows the user to choose an item for deletion. Code both the BeforePicking and AfterPicking events of the ListPicker.

Looks best in full screen.

step 8: replace DisplayList with a more reusable procedure

DisplayList displayed a particular list in a particular label. Replace it with GenerateDisplayText which will generate displayText for any list and can be put in any label.

Looks best in full screen.

step 9: refine the user interface

Add an app icon, descriptive name, and some formatting to the data.

Looks best in full screen.
