How do you program an app’s event-response behavior?

Think of an app like an event-response machine. The user clicks a button, the app responds. A text comes in, the app sends an auto-reply. A flying saucer hits an asteroid, it blows up. Your job as a programmer is to program such event-responses. In App Inventor, you do this directly-- your code is essentially a set of event-responses, of when-do blocks. Check out the following samples:

Example 1. How do you turn a button red when the user clicks it?

OK, this is a really silly app but it will get us started. The app appears and there is a gray button. When the user touches (clicks) the button, the button's color changes to red. The click of the button, Button1.Click, is an event, something that can happen to the app. The gold when Button1.Click block is called an event-handler. The blocks inside it are the response to the event. set means to change something, in this case, change the button’s BackgroundColor to red. The blocks, as a whole, specify that when the user clicks the button it should turn red.

Example 2. How do you say "I’m shaking” when the user shakes the phone?

This app has two components, an AccelerometerSensor and a TextToSpeech. When the user shakes the phone or tablet, the AccelerometerSensor senses the event and the when AccelerometerSensor1.Shaking event handler is triggered. The TextToSpeech component is called to carry out the response of speaking the words “I’m shaking”.
