How do you record information in a list?

Example 1. How do you record notes input by the user?

This behavior illustrates the basics of a note taking or twitter-like app. You first define a variable noteList and initialize it to the create empty list block. This marks the variable noteList as a list so you can put items into it.

When the user enters a note in the NoteTextBox and clicks “Submit”, add items to list is called to append the user’s entry to the end of the list. noteList is then set into ListLabel.Text to display it to the user, and the user’s entry in NoteTextBox is blanked out to ready it for the next entry.

Note: this data in this sample is not persistent, meaning the notes entered by the user will only be recorded for the current run of the app. If you stopped the app and re-opened it, the notes list would be empty. To learn how to record data persistently, see these notes

Example 2. How do you record the phone numbers of incoming SMS texts and then call one randomly to tell them they've won a prize?

This app is great for contests. You ask your audience to text you, then you click a button to pick a winner and the winner’s phone rings!

You first define a variable incomingTexts and initialize it to the create empty list block. This marks the variable as a list so you can put items into it.

When a text comes in, Texting.MessageReceived is triggered. The app first checks to make sure the number isn’t already in the list. If it is not, the number is added to the list.

When ready, the user can click the PickWinnerButton.Click button. In that event-handler, pick a random item is called to to grab a lucky winner and a PhoneCall component is used to call the number.
